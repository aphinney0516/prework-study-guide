Prework Study Guide

HTML

  • The head element contains information, a.k.a meta-data, about the webpage, which is not visible to the user.
  • The body element represents the visible content shown to the user.
  • HTML is made up of elements, and elements are made up of attributes.
  • Attributes provide additional details about each element.
  • Attributes are always defined in the beginning of an element.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Inline CSS only styles one line of code.
  • Internal Style Sheet is a set of style rules for the entire webpage and lives within the head as an element.
  • External Style sheet lives in a separate file and is connected by a "link" element in the head. This is the most commonly used type.
  • Each basic line of code in CSS has a property and a value.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Branches are workspaces you create to make changes to the project without actually affecting it.
  • Repositories, or Repos, are projects contained in GitHub.
  • Markdown (.md) is the way files are formatted in Git.
  • Commit Changes is to save a new version of the branch you're working on.
  • Pull Request is to move the changes on your branch to the main branch, or another branch.
  • Merge is when the changes are pulled to the main branch upon approval.
  • Issue is a task or a fix within the repo.
